# 4. Boas práticas de Compose

- Nomeie composables de forma clara (ex: `TelaLogin`, `BotaoSalvar`)
- Separe partes reutilizáveis da UI em funções isoladas
- Evite colocar lógica complexa dentro dos composables
- Use `LazyColumn` ao invés de `Column` com listas longas
- Prefira `rememberSaveable` para estados persistentes
- Utilize `MaterialTheme` para cores, formas e tipografia consistentes
- Use `Preview` para validar comportamentos visuais:

```kotlin
@Preview
@Composable
fun PreviewLogin() {
    TelaLogin()
}
```

- Utilize `Scaffold` com `TopAppBar` ou `BottomAppBar` para estrutura
- Organize o código de forma modular: telas, componentes, tema, navegação
